# وحدة استيراد البيانات من الأنظمة القديمة

تتيح هذه الأداة نقل بياناتك من أنظمة المحاسبة القديمة (المبنية على SQLite) إلى نظام **قيد** بشكل آمن وكامل — دون الحاجة إلى الإنترنت ودون إرسال أي بيانات لخادم خارجي.

## كيف تعمل الأداة؟

تمرّ عملية الاستيراد بأربع مراحل متتالية:

```
ملف .db  →  تحليل وتحويل  →  حزمة JSON  →  استيراد في التطبيق  →  مراجعة وتأكيد
```

### المرحلة ١ — رفع الملف وتحليله

ترفع ملف قاعدة البيانات (`.db`) من نظامك القديم. تقرأ الأداة الهيكل تلقائياً وتحدد نوعه:

- **الهيكل الأساسي** — يعتمد على جدولَي `customers` و`transactions`.
- **الهيكل المتقدم** — يعتمد على جدول `doc_hdr` الذي يحتوي على معلومات إضافية كالصناديق والحالة.

### المرحلة ٢ — تحويل البيانات

تستخرج الأداة البيانات التالية وتحوّلها إلى صيغة قيد الموحدة:

| البيانات | المصدر | التحويل |
|----------|--------|---------|
| العملات | جدول `currency` | تحديد الكود الدولي (SAR, YER, USD...) |
| المجموعات | جدول `groups` | تصنيف الحسابات |
| الحسابات | جدول `customers` | أطراف (parties) |
| الحركات العادية | `transactions.in = 1/0` | سندات قبض / صرف |
| التحويلات | `transactions.t_cus_id` | سندان (قبض + صرف) عبر الصندوق |

### المرحلة ٣ — تحميل الحزمة (JSON v2.0)

تنتج الأداة ملف JSON منظّماً يحتوي على جميع البيانات المحوّلة. هذا الملف هو **الوسيط** بين نظامك القديم وتطبيق قيد.

### المرحلة ٤ — الاستيراد في التطبيق

يعالج تطبيق قيد الحزمة عبر ثلاث مراحل فرعية:

1. **فرز الحسابات** — يقارن كل حساب قديم بالحسابات الموجودة في قيد (بالاسم والهاتف).
2. **حسم التعارضات** — لكل حساب مكرر تختار: **دمج** أو **إنشاء جديد** أو **تخطي**.
3. **استيراد السندات** — تُنشأ السندات في حالة **انتظار** (مسودة) لمراجعتها وتأكيدها.

## تحويل العملات الذكي

تتعرف الأداة على الأسماء العربية والإنجليزية للعملات الشائعة وتحوّلها تلقائياً:

| الاسم القديم | الكود | الملاحظة |
|--------------|-------|----------|
| سعودي، ريال سعودي | SAR | |
| محلي، يمني | YER | الافتراضي للأنظمة اليمنية |
| دولار، امريكي | USD | |
| مصري، جنيه | EGP | |
| درهم، إماراتي | AED | |
| كويتي | KWD | |
| قطري | QAR | |

> إذا لم تكن العملة موجودة في قيد، يُضيفها النظام تلقائياً عند الاستيراد.

## حركات التحويل بين الحسابات

عندما يحتوي سجل المعاملة على حقل `t_cus_id` (طرف ثانٍ للتحويل)، تعامله الأداة كـ**تحويل** ولا تعامله كقبض أو صرف عادي.

في تطبيق قيد يُترجَم التحويل إلى **سندَّين مترابطين** يمرّان عبر الصندوق:

```
الطرف الأول (cus_id)  →  [سند قبض → الصندوق]  →  [سند صرف → الصندوق]  →  الطرف الثاني (t_cus_id)
```

كلا السندين يبقى في **حالة انتظار** حتى يعتمدهما صاحب الصندوق.

---

## مراجعة التعارضات في التطبيق

عند استيراد الحزمة في تطبيق قيد، تظهر شاشة **حسم التعارضات** لكل حساب:

| الحالة | المعنى | الخيارات المتاحة |
|--------|--------|-----------------|
| ✅ تطابق تام | حساب بنفس الاسم موجود | دمج / إنشاء جديد / تخطي |
| 🔶 تطابق جزئي | حساب بنفس الهاتف موجود | دمج / إنشاء جديد / تخطي |
| ➕ جديد | لا يوجد حساب مطابق | يُنشأ تلقائياً |

بعد حسم التعارضات، تظهر جميع السندات المستوردة في قائمة **انتظار الاعتماد** ليراجعها صاحب الحساب ويوقّعها.

---

## الخطوات بعد تحميل الحزمة

1. افتح تطبيق **قيد** على هاتفك.
2. اذهب إلى **الإعدادات ← استيراد بيانات**.
3. اختر ملف JSON الذي تم تحميله.
4. راجع الحسابات وحسم التعارضات.
5. اعتمد السندات المستوردة من قائمة الانتظار.
